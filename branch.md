# branch
- 기존 브랜치와 같은 내용으로 다른 작업 공간을 만든다.
- 이것을 한 줄기에서 여러개로 뻗어나간다고 해 branch라고 부른다.

# checkout
- 여러 브랜치가 있을 때 다른 브랜치로 옮겨 가는 명령어

# merge 
```
git merge master
```
- 형태로 사용한다.
- 현재 branch에 master 브랜치의 내용을 가져와서 변경사항을 merge 한다.
- 이 때 같은 지점을 수정 했을 경우 merge conflict가 발생 할 수 있다.

# pull request
변경한 내용의 병합을 요청한다.
